# 一文讲透如何构建 Harness——六大组件全解析

> 来源：微信公众号「腾讯云开发者」

## 背景

**Agent = Model + Harness。模型提供智能，Harness 让智能变得有用。**

如果你不是模型本身，那你就是 Harness 的一部分。

2025 年底，一位创业者在社交媒体上发了一条动态：

> "我花了三个月调 Prompt，模型回答质量提升了 20%。然后我花了两周搭 Harness，整体任务完成率从 35% 飙到了 82%。"

过去两年，几乎整个行业都在追逐更大的模型、更强的推理能力。但一个令人尴尬的事实是：绝大多数真实业务场景中，用户并没有感受到与基准分数相匹配的能力提升。

因为大家盯错了地方。

## Agent = Model + Harness

一个真正的 Agent 必须是这样的结构：

**Agent = Model + Harness**

- Model 是大脑，负责"想"
- Harness 是大脑之外的一切——感官系统、运动系统、记忆系统、能量系统——负责让"想"变成"做"

## 裸模型的四个硬伤

### 硬伤一：无法维持跨会话状态

你跟一个裸模型聊了两个小时，讨论了一个复杂的系统架构。然后你关掉窗口，第二天回来——它什么都不记得了。

这不是"记忆力差"的问题，这是**根本没有记忆机制**的问题。

### 硬伤二：无法执行代码

裸模型可以"写"代码，但它不能"跑"代码。它可能写出语法正确但逻辑错误的代码，可能忽略边界条件。而这一切错误，只有在人类手动复制粘贴到本地环境运行之后才会暴露。

### 硬伤三：无法获取实时知识

大语言模型的知识有一个"截止日期"。截止日期之后发生的一切——新发布的 API 文档、最新的安全漏洞、上周刚更新的框架版本——它统统不知道。

### 硬伤四：无法搭建工作环境

裸模型没有文件系统，没有工作空间，没有项目目录。它无法完成任何"工程级"的任务。

## 硬伤与组件对应表

| 硬伤 | 缺失的能力 | Harness 的对应组件 |
|------|------------|---------------------|
| 无法维持跨会话状态 | 长期记忆 | 文件系统 + 记忆（AGENTS.md） |
| 无法执行代码 | 行动能力 | Bash + 沙箱 |
| 无法获取实时知识 | 感知能力 | Web Search + MCP |
| 无法搭建工作环境 | 环境操控 | 文件系统 + 上下文工程 + 编排 |

## 组件一：文件系统——最基础的原语

**文件系统是一切工程活动的地基。**

对于 AI Agent 来说，文件系统的意义甚至比对人类更大。因为人类有大脑记忆，可以"记住"一些中间状态。但 Agent 的上下文窗口是有限的——即使是 200K token 的上下文，对于一个真实项目来说也远远不够。

**文件系统 + Git 给了 Agent "试错"的能力。**

## 组件二：Bash + 沙箱——让 Agent 从"说"到"做"

有了 Bash，Agent 可以：
- 运行它自己写的代码
- 安装依赖包
- 执行构建命令
- 运行测试套件
- 查看运行时错误并调试

**自我验证循环：写→跑→看→修→再来**

实际测试数据表明：在编程任务中，具备自我验证循环的 Agent 的任务完成率比"一次性生成"的方式高出 **40%–60%**。

**为什么必须是"沙箱"？**

沙箱提供了一个隔离的执行环境：
- 资源限制：防止死循环或内存泄漏拖垮系统
- 网络隔离：防止数据泄露或恶意攻击
- 文件系统隔离：Agent 只能访问自己的工作目录
- 超时机制：执行时间超过阈值自动终止

## 组件三：记忆（AGENTS.md）——不改权重也能给模型加知识

AGENTS.md 的设计理念：**工作中写入知识，存文件，下次自动注入。**

写入阶段 → 存储阶段 → 注入阶段

**"上下文注入 = 不改权重给模型加知识"**

传统上，要给模型"加知识"，你需要微调——这个过程成本高、周期长、且存在灾难性遗忘的风险。

AGENTS.md 提供了一条全新的路径。

## 组件四：Web Search + MCP

Web Search 解决了知识截止日期的问题。MCP（Model Context Protocol）则提供了一种标准化的方式，让 Agent 能够连接各种外部工具和数据源。

## 组件五：上下文工程

对抗信息腐烂——让 AI 始终知道"该看什么、该用什么"。

核心思想：**全量灌输不如渐进披露。**

- AGENTS.md 只保留最关键规则
- 详细设计和规范放到 docs/
- 让 Agent 按需查阅，而不是一次塞满

## 组件六：编排 + Hooks

编排负责**组织任务流程**，Hooks 负责**保障执行质量**。

常见 Hook 类型：
- **SessionStart** — 加载上下文、重置风险状态
- **PreToolUse** — 危险动作直接拦截
- **PostToolUse** — 自动注入相关领域规则
- **Stop** — 检查是否有真正完成的证据
- **SessionEnd** — 自动反思、分析行为轨迹

## 总结

**模型已经足够强了，接下来的竞争，越来越多是在"谁更会设计它的运行环境"。**

所以如果非要用一句最短的话来定义 Harness：

**Harness Engineering，不是让模型更聪明，而是让模型真正可驾驭、可上线、可交付。**
