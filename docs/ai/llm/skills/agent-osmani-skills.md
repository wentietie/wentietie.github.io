# Agent Skills：Google 工程师开源的"治懒"技能包

> 来源：微信公众号「开源星探」

## 背景

AI编程工具现在确实能干活了，但用久了你就会发现一个特别烦人的事：它太会给自己找台阶下了。

你说让它写个功能，它啪啪啪给你生成一堆代码，看起来还挺像那么回事。但你想让它先写个测试？它会告诉你"这个逻辑很简单，后面再补"。让它写个文档？它觉得"代码本身就是最好的文档"。

结果就是代码写得飞快，一到上线各种问题全冒出来了。

最近 Google 的一位资深工程师 Addy Osmani 做了一件事，他把自己在 Google 干了多年的工程经验整理成了一套技能包，专门治 AI 编程工具的这个毛病。

## 这套技能包到底干了什么

**一句话概括：把一个资深工程师的肌肉记忆，变成了 AI 必须遵守的规矩。**

现在市面上的 AI 编程工具，Claude Code 也好，Cursor 也好，Copilot 也好，能力都很强，但它们缺少一个东西：**工程纪律**。它们能帮你快速写代码，但不会主动提醒你先写规格、先写测试、先做安全检查。

Addy Osmani 把 Google 内部沉淀的工程实践拆成了 **19 个技能**，覆盖了从需求定义到上线的完整链路：

- **Define** 阶段帮你把需求想清楚
- **Plan** 阶段把任务拆明白
- **Build** 阶段做增量开发
- **Verify** 阶段跑测试、做调试
- **Review** 阶段检查代码质量和安全
- **Ship** 阶段管 Git 工作流和 CI/CD

每个环节都有硬性约束。不是建议，是约束。

比如"Spec before code"，**先写规格再动手，没有规格不准写代码**。"Tests are proof"，**测试不是可选项，是你功能写完的证据**。

## 7 条指令，一条龙打通

除了 19 个技能，还有 **7 条快捷指令**，对应开发周期的 7 个关键节点：

| 指令 | 作用 |
|------|------|
| `/spec` | 让你先把要做什么定义清楚 |
| `/plan` | 帮你规划实现路径 |
| `/build` | 按步骤逐步构建 |
| `/test` | 验证每个功能点 |
| `/review` | 做代码审查 |
| `/code-simplify` | 清理冗余代码 |
| `/ship` | 发布上线 |

你不用记那么多东西，到哪个阶段就用哪条指令，AI 会自动加载对应的技能包。

## 最有意思的部分：反借口机制

这个设计我觉得是最有价值的。

AI 偷懒是有套路的，而且套路就那么几个。Addy 把这些套路全列出来了，然后一个一个给出了堵回去的话术。

**AI 说"这个需求很简单不用写文档"**，技能包里就有对应的反驳：任何需求不论复杂度都必须有书面规格，因为你觉得简单的事情三个月后你也会忘。

**AI 说"测试后面再补"**，技能包告诉你：测试不是补充环节，是功能完成的定义标准。没有测试的功能等于没做完。

这本质上是在跟 AI 的"惰性"做对抗。AI 模型训练时学到了一个倾向：尽快给用户一个看起来能用的结果。这个倾向在写代码的时候特别明显。Agent Skills 做的事情就是用 prompt 工程把这个倾向硬性纠正过来。

## 怎么装

一条命令的事：

```
npx skills add addyosmani/agent-skills
```

装完就能用了。支持 Claude Code、Cursor、Windsurf、GitHub Copilot，基本上主流的 AI 编程工具都兼容。

GitHub 地址：https://github.com/addyosmani/agent-skills

## 写在最后

如果你每天都在用 AI 写代码，但又总觉得写出来的东西差点意思，可能不是 AI 的问题，是你没给它定规矩。这套技能包就是那个规矩。
