# OpenCode 最佳实践

## 概述

OpenCode 作为开源版 Claude Code，在实际使用中有许多可以提升效率的实践方法。本文汇总社区经验，帮助你更好地使用 OpenCode。

## 任务分层策略

### 大任务 vs 小任务

OpenCode 适合处理复杂任务，但对于不同规模的任务应采用不同策略：

**大任务（从 0 到 1 搭建）**：
- 让 OpenCode 先做整体规划
- 分步骤确认后再执行
- 适合使用 Oh-My-OpenCode 多 Agent 模式

**小任务（局部模块修改）**：
- 直接描述需求
- 单次交互完成
- 避免过度设计

## 模型选择策略

OpenCode 支持 75+ 模型，选择合适的模型事半功倍：

### 主代理推荐

| 模型 | 适用场景 | 特点 |
|------|----------|------|
| glm-5 | 通用编程 | 平衡效果好 |
| deepseek-reasoner | 复杂推理 | 思考深入 |
| kimi-2.5 | 快速任务 | 响应快 |

### 配置示例

```json
{
  "model": "glm-5",
  "temperature": 0.7,
  "max_tokens": 4096
}
```

## Oh-My-OpenCode 高效使用

Oh-My-OpenCode 是 OpenCode 的核心插件，熟练使用能大幅提升效率。

### 基础配置

```json
{
  "plugins": ["oh-my-opencode"],
  "agent": {
    "model": "glm-5",
    "loop": "raph"
  }
}
```

### 循环模式选择

- **ulw**：适合需要快速迭代的场景
- **raph**：适合需要深度思考的任务
- **default**：适合一般任务

### Token 节省技巧

1. **明确任务边界**：减少不必要的上下文
2. **分步执行**：大任务拆分为小步骤
3. **选择合适的模型**：不必每个任务都用最强大的模型

## 项目集成最佳实践

### 1. 项目初始化

首次在项目中使用 OpenCode：

```
你现在在我的项目根目录。

请快速阅读 README、package.json 和关键入口文件，给出：
- 技术栈
- 目录结构说明
- 我应该从哪个入口理解业务
```

### 2. 功能开发模板

```
需求如下：...

约束如下：...

请先给实现计划（不要改代码）。
我确认后你再修改，并展示 diff。
```

### 3. Bug 修复模板

```
我遇到 bug：...

复现步骤：...

错误日志：...

请定位原因 → 提供修复 → 补测试 → 给出验证方式。
```

### 4. 重构任务模板

```
现在 src/utils/date.ts 有很多重复逻辑。

请重构成更清晰可维护的版本，保证所有单测仍通过。
```

## 版本控制集成

### 分支管理

让 OpenCode 自动管理 Git：

```
请按最佳实践新建一个 feature 分支，完成修改后总结变更，并给出 PR 描述。
```

### Commit 规范

```
[feat] 添加用户认证功能
[fix] 修复登录超时问题
[refactor] 重构数据库连接模块
```

## 性能优化

### 1. 项目理解一次，多次复用

首次让 OpenCode 全面了解项目，后续交互会更高效。

### 2. 缓存常用上下文

对于需要反复使用的项目结构信息，可以保存为项目文档。

### 3. 合理使用 @ 引用

- `@file` 引用单个文件
- `@dir` 引用目录
- 避免无必要的全目录扫描

## 安全注意事项

### 1. API Key 管理

- 不要将 API Key 提交到代码仓库
- 使用环境变量管理敏感信息

### 2. 权限控制

- 生产环境谨慎操作
- 重要文件修改前先确认

### 3. 代码审查

- AI 生成的代码需要人工审查
- 不要盲目信任 AI 输出

## 常见坑与避坑指南

| 场景 | 坑点 | 避坑方法 |
|------|------|----------|
| 编码风格 | 易写出不符合项目习惯的代码 | 先提供项目规范和示例 |
| 代码分层 | 可能忽略现有分层架构 | 明确告知现有架构 |
| 功能组合 | 可能过度设计 | 控制任务边界 |
| Token 消耗 | 大任务容易超支 | 分步执行，及时止损 |

## 进阶用法

### 多 Agent 协作

对于超大型任务，可以设计多 Agent 工作流：

1. **规划 Agent**：分解任务，制定计划
2. **执行 Agent**：按计划执行具体编码
3. **审查 Agent**：检查代码质量和规范

### MCP 工具扩展

结合 MCP (Model Context Protocol) 扩展 OpenCode 能力：

```javascript
// 配置示例
{
  "mcpServers": {
    "filesystem": true,
    "docker": true
  }
}
```
